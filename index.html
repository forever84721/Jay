05 — infrastructure

部署與維運

我不只寫 code，也負責讓它上線並持續運作。從 CI/CD 自動化流程，到容器化部署、反向代理與 TLS。

code→GitHub→Actions · build + test→docker→Linux · Cloudflare→live

GitHub Actions CI/CD

以 GitHub Actions 建立自動化 pipeline，push 後自動 build、跑測試、封裝 Docker image 並部署至自架 Linux 伺服器（self-hosted runner）。

容器化部署

用 Docker 建置與打包環境（SQL Server、應用服務），讓開發到上線環境一致、可重現。

反向代理 / TLS / Cloudflare

Linux server 上以 Nginx 反向代理多服務，搭配 Cloudflare 處理 DNS、TLS 與 CDN。

Windows 桌面部署

POS 系統以 WinForm 打包部署至客戶端 Windows 環境，處理 Windows 相容性差異、SerialPort 驅動問題，以及門市端的版本更新與維護流程。

上線維運

POS 系統已上線數十家門市、健康類產品持續服務使用者——長期維運與迭代的實戰經驗。
